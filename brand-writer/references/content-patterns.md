# Content Patterns by Type

## Landing Page Headlines

### Structure
**Primary**: Outcome or transformation statement (5-10 words)
**Secondary**: How we enable it or what it means (10-20 words)

### Patterns

**The Question Opener**
Ask the core problem, then answer it.
```
Who has access to what—and should they?
The trust layer for agents that actually work.
```

**The Paradigm Shift**
Name the old way, introduce the new.
```
From demo mode to production.
Give your agents the governance they need to ship.
```

**The Speed Promise**
Compress time dramatically.
```
Zero to governed agent in 10 minutes.
Connect. Observe. Control. Ship.
```

**The Bold Claim**
State the future we're building.
```
The trust layer of the agentic web.
Identity, auth, and context for agents that run for real.
```

### Anti-patterns to Avoid
- Generic: "The Future of Security" (says nothing)
- Fear-based: "Don't Let Agents Go Rogue"
- Buzzword soup: "AI-Powered Next-Gen Security Platform"
- Too clever: Puns or wordplay that obscure meaning

## Product/Feature Descriptions

### Structure
1. **What it does** (1 sentence, outcome-focused)
2. **How it works** (1-2 sentences, mechanism)
3. **Why it matters** (1 sentence, benefit)

### Example
```
**OAuth Connector**
Connect your agent to GitHub, Slack, Linear, and more—with hosted OAuth that handles tokens for you.
No redirect flows to build. No secrets to manage. Just pick the tool, authenticate once, and your agent gets scoped access.
Ship integrations in hours, not weeks.
```

### Tone Calibration
- **For developers**: Be precise. Name technologies. Show code snippets.
- **For security teams**: Emphasize control, auditability, policy enforcement.
- **For executives**: Lead with business outcomes and risk reduction.

## Hero Sections

### Components
1. **Headline**: The hook (see patterns above)
2. **Subhead**: Expand or ground the claim (1-2 lines)
3. **CTA**: Action-oriented, specific ("Start building" > "Learn more")
4. **Social proof**: Logos, metrics, or quote (optional but powerful)

### Example
```
Headline: The command center for your agents.
Subhead: See what they're doing. Decide what they're allowed to do. Ship with confidence.
CTA: Start your dev preview
Proof: Trusted by teams building the agentic future.
```

## Blog Posts / Thought Leadership

### Opening Patterns

**The Observation**
Start with something true that readers recognize.
```
Most agent demos work. Most agent deployments don't.
The gap isn't intelligence—it's infrastructure.
```

**The Contrarian Take**
Challenge a common assumption.
```
"Just use OAuth" is the worst advice in agent development.
Here's why—and what to do instead.
```

**The Future Frame**
Paint a picture of what's coming.
```
In two years, every SaaS product will have an agent mode.
The question isn't if—it's whether those agents will be governable.
```

### Structure
1. **Hook**: First 2-3 sentences must earn attention
2. **Problem**: What's broken or changing
3. **Insight**: The non-obvious reframe
4. **Solution**: How to think about or address it
5. **Close**: Call to action or thought to carry away

### Voice in Thought Leadership
- More personal ("we've seen," "our bet")
- Take positions (don't hedge everything)
- Teach something useful even to non-customers
- Link to product naturally, not forcefully

## Marketing Emails

### Subject Lines
- Keep under 50 characters
- Be specific over clever
- Create curiosity or promise value

**Good**: "Your agent's first governed call"
**Bad**: "Exciting news from Kontext!"

### Body Structure
1. **Opening**: Personal, relevant, brief (1-2 sentences)
2. **Value**: What's new or why it matters (2-3 sentences)
3. **CTA**: Single, clear action
4. **Sign-off**: Human, warm

### Example
```
Subject: You're in the dev preview

Hey [Name],

Your Kontext dev preview is live. You can now connect your first tool,
make a governed agent call, and see everything in one dashboard.

Here's the fastest way to start:
→ [Connect GitHub in 2 minutes]

If you hit any friction, reply to this email. We read everything.

— Jens
```

## CTAs (Calls to Action)

### Hierarchy
- **Primary**: Start action ("Start building," "Get dev preview")
- **Secondary**: Learn more ("See how it works," "Read the docs")
- **Tertiary**: Social proof ("See customer stories")

### Patterns
- Use verbs: "Start," "Get," "Try," "See," "Build"
- Be specific: "Deploy your first agent" > "Get started"
- Match commitment level to page depth

### Avoid
- Vague: "Learn more," "Click here"
- Pushy: "Buy now," "Don't miss out"
- Passive: "Information available"

## Taglines and Slogans

### Current
- "The trust layer of the agentic web"
- "Make AI execution safe, visible, and honest"

### Creating New Ones
1. Capture one essential truth
2. Use concrete words over abstractions
3. Be memorable through rhythm or surprise
4. Test: Can someone repeat it after hearing once?

### Format Options
- **Noun phrase**: "The trust layer of the agentic web"
- **Imperative**: "Ship agents that actually work"
- **Promise**: "From demo to production in minutes"
